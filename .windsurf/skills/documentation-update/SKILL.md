# 文档更新工作流

完成功能开发或问题修复后，系统化更新项目文档的工作流程。

## 核心原则

1. **及时更新** - 完成变更后立即更新文档
2. **保持一致** - 文档内容与代码实现同步
3. **简洁明了** - 避免冗余，突出关键信息
4. **结构清晰** - 遵循统一的文档格式

## 触发条件

| 场景 | 优先级 | 必须更新的文档 |
|------|--------|----------------|
| 新功能开发 | 高 | 笔记、README、规则文件 |
| Bug 修复 | 中 | 笔记、变更日志 |
| 代码重构 | 中 | 笔记、规则文件 |
| 新增组件/模块 | 高 | 笔记、README、API 文档 |
| 架构变更 | 高 | 笔记、规则文件、README |
| 依赖更新 | 中 | 笔记、构建文档 |
| 性能优化 | 中 | 笔记、性能文档 |

## 执行流程

### 第一步：创建/更新开发笔记

在 `docs/Notes/` 目录下创建或更新笔记文件。

#### 文件命名规范

| 类型 | 命名格式 |
|------|----------|
| 功能开发 | `<feature-name>-implementation-notes.md` |
| Bug 修复 | `<issue-name>-fix-notes.md` |
| 重构 | `<refactor-name>-notes.md` |
| 性能优化 | `<optimization-name>-notes.md` |

#### 笔记模板

```markdown
# <标题>

## 概述

**创建日期**: YYYY-MM-DD  
**相关 Issue**: #xxx（如有）

---

## 一、变更概述

### 新增文件
| 文件路径 | 功能描述 |
|----------|----------|
| `path/to/file.cpp` | 功能说明 |

### 修改文件
| 文件路径 | 修改内容 |
|----------|----------|
| `path/to/file.cpp` | 修改说明 |

---

## 二、实现的功能特性

- ✅ 功能点 1：详细描述
- ✅ 功能点 2：详细描述

---

## 三、技术实现细节

### 关键代码片段

### 设计决策

### 数据流

---

## 四、遇到的问题及解决方案

---

## 五、测试验证

| 场景 | 预期结果 | 实际结果 |
|------|----------|----------|
| 场景 1 | ... | ✅ 通过 |

---

## 六、后续工作

- [ ] 待优化项
```

### 第二步：更新变更日志

检查并更新 `CHANGELOG.md`（如存在）。

```markdown
## [Unreleased]

### Added
- 新增功能描述

### Changed
- 变更描述

### Fixed
- 修复描述
```

### 第三步：检查规则文件

检查 `.windsurf/rules/` 目录下的规则文件是否需要更新。

| 规则文件 | 更新条件 |
|----------|----------|
| `01-project-overview.mdc` | 新增技术或目录结构变化 |
| `02-architecture.mdc` | 架构变更、新增模块 |
| `03-cpp-standards.mdc` | 新增 C++ 编码模式 |
| `04-qml-standards.mdc` | 新增 QML 组件模式 |
| `05-build-system.mdc` | 构建配置变更、新增依赖 |
| `06-i18n.mdc` | 新增翻译规则 |
| `07-performance.mdc` | 性能相关变更 |
| `08-ui-design.mdc` | UI 设计规范变更 |
| `09-shader-standards.mdc` | Shader 变更 |
| `10-skills-and-tools.mdc` | 新增技能或 MCP 服务 |

### 第四步：更新 README 文件

| 检查项 | 更新条件 |
|--------|----------|
| 项目概述 | 功能描述变化 |
| 技术栈 | 新增技术 |
| 目录结构 | 新增目录 |
| 安装/构建说明 | 新增依赖或步骤 |

### 第五步：更新国际化文件（如适用）

```powershell
# 更新翻译源文件
cmake --build . --target EnhanceVision_lupdate
# 编译翻译文件
cmake --build . --target EnhanceVision_lrelease
```

## 输出清单

### 必须完成
- [ ] 开发笔记已创建/更新（`docs/Notes/`）
- [ ] 规则文件已检查并按需更新（`.cursor/rules/`）

### 按需完成
- [ ] 变更日志已更新（重大变更时）
- [ ] README 文件已更新（结构/功能变更时）
- [ ] 国际化文件已更新（文本变更时）

## 文档质量检查

- [ ] 代码示例可运行
- [ ] 所有路径正确
- [ ] 日期格式统一（YYYY-MM-DD）
- [ ] Markdown 语法正确，代码块有语言标记
- [ ] 中文文档使用中文，术语翻译一致
