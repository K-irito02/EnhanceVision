# 规则治理总则

## 规则优先级

1. `using-superpowers` 技能（始终优先）
2. `00-rule-governance.md`（治理总则）
3. `02-architecture.md`（架构边界）
4. 语言规范（`03`、`04`）
5. 领域规范（`05`~`09`）
6. 工具流程（`10`）

## 单一职责边界

| 文件 | 职责 |
|------|------|
| `01` | 项目事实基线（技术栈、目录） |
| `02` | 分层职责与数据流 |
| `03` | C++ 编码规范 |
| `04` | QML/JS 编码规范 |
| `05` | 构建与部署流程 |
| `06` | 国际化 |
| `07` | 性能与线程 |
| `08` | UI 设计系统 |
| `09` | Shader 规范 |
| `10` | 技能与工具使用 |

## 冲突处理

- 条款冲突时：优先"边界规则"与"更具体的文件"
- 新规则必须避免与现有规则重复
- 修改规则时优先"合并到职责文件"，禁止横向复制

## 变更要求

- 规则变更需说明：原因、影响范围、迁移建议
- 功能完成后同步更新 `docs/Notes/` 与相关规则
- 新增技能时必须更新 `10-skills-and-tools.md`
