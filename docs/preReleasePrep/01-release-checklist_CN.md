# 发布检查清单

[English](01-release-checklist_EN.md) | 简体中文

本文档列出了每次发布前必须完成的检查项目，确保发布质量。

## 📋 发布前检查清单

### 一、代码质量检查

#### 1.1 编译检查
- [ ] Release 模式编译无错误
- [ ] Release 模式编译无警告（或已知警告已记录）
- [ ] 所有单元测试通过
- [ ] 静态代码分析通过（如有配置）

```powershell
# 执行编译检查
cmake --preset windows-msvc-2022-release
cmake --build build/msvc2022/Release --config Release -j 8

# 运行测试
ctest --test-dir build/msvc2022/Release -C Release --output-on-failure
```

#### 1.2 代码审查
- [ ] 所有代码变更已通过 Pull Request 审查
- [ ] 无 TODO 或 FIXME 标记（或已记录为已知问题）
- [ ] 代码格式符合规范
- [ ] 无硬编码的调试代码或测试数据

#### 1.3 安全检查
- [ ] 无敏感信息泄露（API 密钥、密码等）
- [ ] 无已知安全漏洞
- [ ] 第三方依赖版本无已知安全问题

## ⚠️ 常见问题

### Q: 编译警告是否必须全部修复？
A: 建议修复所有警告。如果某些警告无法修复，需要在发布说明中记录。

### Q: 测试覆盖率达到多少才算合格？
A: 核心功能必须 100% 测试通过，边缘功能可以记录为已知问题。

### Q: 发布后发现 Bug 怎么办？
A: 根据严重程度决定：
- 严重 Bug：立即发布补丁版本（PATCH）
- 一般 Bug：记录到 Issues，下个版本修复
- 小问题：记录到已知问题列表
