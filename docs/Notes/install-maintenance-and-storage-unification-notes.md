# 安装维护与存储路径收口说明

## 概述

本次调整将旧数据“迁移/清理”从设置页移出，统一收敛到安装器升级流程和应用首启维护链路中处理。

后续补充修正了升级后缓存统计目录探测的兜底链路，并清理了首启/关闭过程里无诊断价值的运行日志输出，避免普通场景产生噪声告警。

## 关键变更

- 配置根目录统一为 `%LOCALAPPDATA%\EnhanceVision`
- `settings.ini`、`ui_state.json`、`sessions.json` 不再分散落盘
- 安装器升级页新增旧数据处理动作：保留、迁移、删除
- 安装器只写入维护意图，不再直接做业务层路径修复
- 应用在日志、缩略图库、UI 状态和会话恢复初始化前，先执行安装维护
- 设置页移除旧数据“迁移”“清理”按钮
- 运行期修改应用数据目录时改为立即执行受控迁移

## 结果

- 升级后首启能按用户在安装器中的选择恢复或清空会话、消息和界面状态
- `sessions.json` 与 `ui_state.json` 不再因为历史路径分散而导致升级后界面不一致
- 旧路径迁移失败时会回退到安全状态，避免进入半初始化或半迁移状态
- 升级后“缓存管理”统计会跟随真实生效数据目录，不会因错误旧路径探测而显示为 `0 B`
- 设置页已移除旧数据“迁移”“清理”按钮，相关操作统一由安装器和首启维护链路接管

## 验证补充

- `InstallMaintenanceServiceTest` 已调整为按测试函数与进程号隔离运行目录，避免重复执行时清理互相干扰
- Release 构建与安装器打包流程已重新验证，当前工作树中的维护链路和文档说明保持一致
- 旧数据目录与旧 metadata 的清理现在按 best-effort 处理，不再因为本地文件系统临时占用阻断升级主流程
