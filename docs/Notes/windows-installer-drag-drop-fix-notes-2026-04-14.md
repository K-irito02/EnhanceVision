# Windows 安装版拖拽导入修复说明（强化版）

## 概述

修复安装版 EnhanceVision 在 Windows 上“安装完成页立即启动后无法拖拽导入媒体文件”的问题，并补齐提权异常场景下的启动自愈逻辑。

## 问题现象

- 安装完成页勾选“立即运行”后，拖拽文件到主页面无反应
- 拖拽覆盖层不出现，说明外部拖拽事件未进入 QML `DropArea`
- 从常规快捷方式启动时问题可弱化或不稳定复现

---

## 根因分析

问题核心不在 QML 业务逻辑，而在“首启进程完整性级别”：

1. 安装器为 `RequestExecutionLevel admin`，完成页启动容易落入提权上下文  
2. 普通权限 Explorer 到高完整性应用的拖拽链路会被系统安全边界拦截  
3. 仅依赖 `ChangeWindowMessageFilter(Ex)` 不是充分条件，必须同时保证首启运行在普通用户上下文

---

## 实现方案

### 1) 安装器启动链路修复（根因位）

- 完成页启动由直接执行改为通过 `explorer.exe` 代启 `EnhanceVision.exe`
- 在保留管理员安装和 HKLM 写入逻辑前提下，确保首启实际运行在用户 shell 上下文

### 2) 应用启动自愈兜底

- `main.cpp` 启动早期检测当前进程是否 elevated
- 若 elevated：自动尝试通过 Explorer 中转重启同一可执行文件（附带防重入参数）
- 若降权重启失败：弹出明确告警并输出 warning，避免用户“静默失败”

### 3) 拖拽接收链路加固

- 主页面入口统一过滤无效 URL、非本地 URL、空路径
- `FileController::addFiles` 再做一次防御性过滤与去重
- 保持业务处理落点仍为 `fileController.addFiles(...)`，避免多处分叉

### 4) 窗口级配置时机兜底

- `Application` 在窗口句柄创建后、窗口接管后、窗口显示时都确保执行拖拽放行配置
- 降低无边框窗口接管与平台事件时序变化带来的偶发失效风险
