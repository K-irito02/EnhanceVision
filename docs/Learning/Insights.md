在 Vibe Coding 中，Qt 项目建议重点关注以下关键点（仅提纲）：

1. 国际化（`tr` / `qsTr` / 翻译资源维护）
2. 亮暗主题（主题切换、统一色板、组件一致性）
3. UI/UX 设计规范（信息层级、交互反馈、空状态/异常状态）
4. 响应式布局（窗口缩放、自适应面板、不同分辨率适配）
5. 国际化对 UI 的影响（文本长度、换行、布局膨胀）
6. 多线程编程（UI 主线程隔离、耗时任务异步化、线程安全）
7. 性能优化（QML 渲染、列表性能、图像加载、对象生命周期）
8. 清理遗留代码与技术债（废弃接口、重复逻辑、无用资源）
9. 跨平台适配（路径、编码、字体、窗口行为、部署差异）
10. 技术栈与第三方库（版本管理、兼容性、许可证、升级策略）
11. 项目架构（总体架构、前端架构、后端架构、模块边界）
12. 设计原则（高内聚低耦合、单一职责、可测试性、可维护性）
13. 项目结构（`src/`、`qml/`、`resources/`、`docs/`、`tests/`）
14. 后端规范（命名、文件头、代码风格、注释、现代 C++、内存管理）
15. 后端质量要求（错误处理、线程安全、日志、QML 集成规范）
16. 前端规范（命名、属性绑定、信号处理、组件拆分、资源引用）
17. 前端质量要求（布局规范、动画规范、JavaScript 使用边界、性能规范）
18. 核心约束（架构约束、性能约束、开发约束、文件约束）
19. 构建与发布（CMake、打包部署、调试/发布配置）
20. 测试与验证（单元测试、集成测试、关键流程回归）
21. 日志与问题定位（分类日志、调试信息、可追踪性）
22. 文档同步（设计文档、接口文档、规则文档及时更新）
23. 可用技能（构建修复、性能优化、QML/C++ 联动、文档更新）
24. 可用 MCP（文档检索、仓库协作、辅助分析能力）
