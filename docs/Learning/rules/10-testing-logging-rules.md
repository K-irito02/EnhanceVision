# 10 测试与日志规范

## 测试覆盖

- 新功能至少覆盖关键主流程验证
- Bug 修复必须补最小复现与回归验证
- 关键模块（处理链路、会话、导出）优先保障

## 回归策略

- UI、处理参数、导出结果保持一致性检查
- 国际化、主题切换、跨模块联动需抽查

## 日志规范

- 日志分层：调试/信息/警告/错误
- 错误日志包含上下文（模块、输入、结果）
- 避免刷屏与无意义重复日志
- 运行日志默认只保留警告、错误和致命问题，信息级输出只在排障时临时开启
- UI 生命周期、窗口缩放、初始化探针等路径中的 `qInfo()` / `qDebug()` 仅用于临时排障，合并前必须移除或降级
- 图片/缩略图/媒体查看器加载链路必须严格区分图像与视频源，避免把视频文件误交给 `Image`/`Loader` 触发可避免的 warning
- 构建日志中的第三方告警要标记来源，避免与项目自身问题混淆

## 问题闭环

- 问题需可复现、可定位、可验证修复
- 修复后记录到对应文档或笔记
