# 多媒体处理流程分析

## 概述

EnhanceVision 支持两种处理模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **Shader 模式** | GPU 着色器滤镜处理 | 实时预览、快速调色 |
| **AI 推理模式** | NCNN 深度学习推理 | 超分辨率增强 |

---

## 一、Shader 模式处理流程

### 1.1 图片处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    Shader 图片处理流程                           │
├─────────────────────────────────────────────────────────────────┤
│  输入图像: QImage (任意尺寸)                                      │
│      ↓                                                          │
│  [Step 1] 加载图像                          ~10-50ms            │
│      ↓                                                          │
│  [Step 2] 格式转换 (→ RGB888)               ~5-20ms             │
│      ↓                                                          │
│  [Step 3] 色彩调整阶段                      ~50-200ms           │
│      ├── 3.1 曝光 (Exposure)                                    │
│      ├── 3.2 亮度 (Brightness)                                  │
│      ├── 3.3 对比度 (Contrast)                                  │
│      ├── 3.4 饱和度 (Saturation)                                │
│      ├── 3.5 色相 (Hue)                                         │
│      ├── 3.6 伽马 (Gamma)                                       │
│      ├── 3.7 色温 (Temperature)                                 │
│      └── 3.8 色调 (Tint)                                        │
│      ↓                                                          │
│  [Step 4] 效果处理阶段                      ~100-500ms          │
│      ├── 4.1 高光/阴影 (Highlights/Shadows)                     │
│      ├── 4.2 晕影 (Vignette)                                    │
│      ├── 4.3 降噪 (Denoise) - 中值滤波                          │
│      ├── 4.4 模糊 (Blur) - 高斯模糊                             │
│      └── 4.5 锐化 (Sharpen) - 拉普拉斯锐化                      │
│      ↓                                                          │
│  [Step 5] 保存结果                          ~20-100ms           │
│      ↓                                                          │
│  输出图像: QImage (相同尺寸)                                      │
└─────────────────────────────────────────────────────────────────┘
```

**处理阶段说明：**

| 阶段 | 进度范围 | 主要操作 |
|------|---------|----------|
| 加载图像 | 0% - 15% | QImage 加载、格式验证 |
| 预处理 | 15% - 20% | 格式转换、内存分配 |
| 色彩调整 | 20% - 60% | 曝光、亮度、对比度、饱和度、色相、伽马、色温、色调 |
| 效果处理 | 60% - 85% | 高光阴影、晕影、降噪、模糊、锐化 |
| 保存结果 | 85% - 100% | 图像编码、文件写入 |

**Shader 参数列表：**

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| brightness | -1.0 ~ 1.0 | 0.0 | 亮度 |
| contrast | 0.0 ~ 3.0 | 1.0 | 对比度 |
| saturation | 0.0 ~ 3.0 | 1.0 | 饱和度 |
| hue | -0.5 ~ 0.5 | 0.0 | 色相偏移 |
| sharpness | 0.0 ~ 2.0 | 0.0 | 锐化强度 |
| blur | 0.0 ~ 1.0 | 0.0 | 模糊程度 |
| denoise | 0.0 ~ 1.0 | 0.0 | 降噪强度 |
| exposure | -2.0 ~ 2.0 | 0.0 | 曝光补偿 |
| gamma | 0.3 ~ 3.0 | 1.0 | 伽马校正 |
| temperature | -1.0 ~ 1.0 | 0.0 | 色温 |
| tint | -1.0 ~ 1.0 | 0.0 | 色调 |
| vignette | 0.0 ~ 1.0 | 0.0 | 晕影效果 |
| highlights | -1.0 ~ 1.0 | 0.0 | 高光调整 |
| shadows | -1.0 ~ 1.0 | 0.0 | 阴影调整 |

### 1.2 视频处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    Shader 视频处理流程                           │
├─────────────────────────────────────────────────────────────────┤
│  输入视频: 任意格式 (MP4/AVI/MOV 等)                              │
│      ↓                                                          │
│  [Step 1] 打开文件                          ~50-200ms           │
│      ↓                                                          │
│  [Step 2] 初始化解码器                      ~100-300ms          │
│      ├── 检测视频编码格式                                        │
│      ├── 获取视频参数 (分辨率、帧率、时长)                        │
│      └── 初始化 FFmpeg 解码上下文                                │
│      ↓                                                          │
│  [Step 3] 初始化编码器                      ~100-300ms          │
│      ├── 配置输出编码器 (H.264/H.265)                            │
│      ├── 设置输出参数 (分辨率、比特率)                            │
│      └── 初始化 FFmpeg 编码上下文                                │
│      ↓                                                          │
│  ╔═══════════════════════════════════════════════════════════╗ │
│  ║  [Step 4] 逐帧处理循环 (占总时间 95%)                        ║ │
│  ║  ┌─────────────────────────────────────────────────────┐   ║ │
│  ║  │ 4.1 解码帧 (FFmpeg)            ~5-20ms             │   ║ │
│  ║  │ 4.2 格式转换 (YUV → RGB888)    ~2-5ms              │   ║ │
│  ║  │ 4.3 应用 Shader 滤镜           ~10-50ms            │   ║ │
│  ║  │ 4.4 格式转换 (RGB888 → YUV)    ~2-5ms              │   ║ │
│  ║  │ 4.5 编码帧 (FFmpeg)            ~5-20ms             │   ║ │
│  ║  └─────────────────────────────────────────────────────┘   ║ │
│  ║  单帧处理耗时: ~25-100ms × 总帧数                           ║ │
│  ╚═════════════════════════════════════════════════════════════╝│
│      ↓                                                          │
│  [Step 5] 完成编码                          ~100-500ms          │
│      ├── 写入文件尾                                              │
│      └── 复制音频流 (如启用)                                     │
│      ↓                                                          │
│  输出视频: MP4 (H.264/H.265 编码)                                │
└─────────────────────────────────────────────────────────────────┘
```

**处理阶段说明：**

| 阶段 | 进度范围 | 主要操作 |
|------|---------|----------|
| 打开文件 | 0% - 5% | FFmpeg 打开输入文件 |
| 解码初始化 | 5% - 10% | 检测格式、初始化解码器 |
| 编码初始化 | 10% - 15% | 配置编码器、分配资源 |
| 帧处理 | 15% - 90% | 解码 → Shader处理 → 编码 |
| 完成编码 | 90% - 95% | 写入文件尾、释放资源 |
| 写入完成 | 95% - 100% | 关闭文件句柄 |

**视频处理特点：**
- 保留原始音频流（可配置）
- 支持 H.264/H.265 编码输出
- 逐帧处理确保效果一致性
- 支持取消操作

### 2.1 CPU 后端处理流程

#### 2.1.1 图片处理流程 (CPU)

```
┌─────────────────────────────────────────────────────────────────┐
│                AI 推理图片处理流程 (CPU)                          │
├─────────────────────────────────────────────────────────────────┤
│  输入图像: 1920 x 1080                                           │
│      ↓                                                          │
│  [Step 1] 模型加载                          ~500-2000ms         │
│      ├── 加载 .param 文件 (网络结构)                             │
│      └── 加载 .bin 文件 (模型权重)                               │
│      ↓                                                          │
│  [Step 2] 预处理                            ~10-50ms            │
│      ├── 格式转换 (→ RGB888)                                    │
│      ├── 归一化 (mean/scale)                                    │
│      └── QImage → ncnn::Mat                                     │
│      ↓                                                          │
│  ╔═══════════════════════════════════════════════════════════╗ │
│  ║  [Step 3] 核心推理 (CPU 串行计算)                           ║ │
│  ║  ┌─────────────────────────────────────────────────────┐   ║ │
│  ║  │ 模式判断:                                            │   ║ │
│  ║  │ - 小图 (≤ tileSize): processSingle()                │   ║ │
│  ║  │ - 大图 (> tileSize): processTiled() 分块处理        │   ║ │
│  ║  └─────────────────────────────────────────────────────┘   ║ │
│  ║                                                            ║ │
│  ║  processSingle() 单次推理:                                  ║ │
│  ║  ├── 创建 Extractor                                    ~1ms║ │
│  ║  ├── 设置输入 blob                                     ~1ms║ │
│  ║  ├── ex.extract() 执行推理                          ~5-30s  ║ │
│  ║  └── 获取输出 blob                                     ~1ms║ │
│  ║                                                            ║ │
│  ║  processTiled() 分块推理 (大图自动启用):                    ║ │
│  ║  ├── 计算分块数量: ceil(W/160) × ceil(H/160)              ║ │
│  ║  ├── 每个分块: 提取 → 推理 → 合并                         ║ │
│  ║  └── 总耗时: 单次推理 × 分块数量                           ║ │
│  ╚═════════════════════════════════════════════════════════════╝│
│      ↓                                                          │
│  [Step 4] 后处理                            ~10-50ms            │
│      ├── ncnn::Mat → QImage                                     │
│      ├── 反归一化                                               │
│      └── 可选: 自定义输出缩放                                   │
│      ↓                                                          │
│  [Step 5] 保存结果                          ~50-200ms           │
│      ↓                                                          │
│  输出图像: 7680 x 4320 (4x 放大)                                 │
└─────────────────────────────────────────────────────────────────┘
```

**CPU 推理性能估算：**

| 输入尺寸 | 处理模式 | 分块数量 | 预计耗时 |
|---------|---------|---------|---------|
| 640 x 480 | 单图 | 1 | ~5-15s |
| 1280 x 720 | 单图 | 1 | ~15-60s |
| 1920 x 1080 | 分块 | ~48 | ~1-5min |
| 3840 x 2160 | 分块 | ~192 | ~5-20min |

## 三、处理模式对比

### 3.1 功能对比

| 特性 | Shader 模式 | AI 推理模式 |
|------|------------|------------|
| 处理速度 | 快 (~ms级) | 慢 (~s~min级) |
| 输出尺寸 | 不变 | 放大 (4x) |
| 效果类型 | 色彩/滤镜调整 | 超分辨率增强 |
| GPU 加速 | 始终使用 | 可选 (Vulkan) |
| 适用场景 | 实时预览、快速调色 | 高质量放大 |

### 3.2 后端选择建议

| 场景 | 推荐后端 | 原因 |
|------|---------|------|
| 快速预览 | Shader 模式 | 实时响应 |
| 小图处理 (< 1MP) | CPU | 开销小，无需 GPU |
| 大图处理 (> 1MP) | GPU | 并行加速明显 |
| 视频处理 | GPU | 逐帧处理耗时大 |
| 无 GPU 环境 | CPU | 兼容性好 |

## 四、关键参数说明

### 4.1 分块处理参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| tileSize | 分块大小 (像素) | 模型定义 |
| tilePadding | 分块重叠区域 | 10-64 (动态) |
| layerCount | 模型层数 | 加载时获取 |

**动态 Padding 计算：**
- layerCount > 500: padding = 64
- layerCount > 200: padding = 48
- layerCount > 50: padding = 24
- 否则: padding = 10

### 4.2 后端配置参数

| 参数 | CPU 模式 | GPU 模式 |
|------|---------|---------|
| num_threads | 1 | 1 |
| use_vulkan_compute | false | true |
| use_packing_layout | false | false |
| lightmode | false | false |
