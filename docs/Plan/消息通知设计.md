## 设计方案

### 1. 会话标签后台任务指示器增强

**现有问题**：
- 只有橙色闪烁一种状态
- 没有区分"正在处理"和"等待处理"
- 主题切换时可能显示异常

**改进方案**：
- **蓝色指示器**：正在处理多媒体文件（活跃状态）
- **灰色指示器**：等待处理多媒体文件（队列状态）
- 增加主题适配，确保在暗色/亮色主题下都清晰可见

### 2. 底部通知栏系统设计

**布局设计**：
```
[左侧状态区域] -------- [右侧通知按钮]
┌─────────────────┐     ┌─────────────────┐
│ • 处理中: 2个   │     │ 🔔 通知 (3)    │
│ • 队列中: 1个   │     │ ⚙️ 设置        │
│ • 已完成: 5个   │     │ ↩️ 恢复        │
└─────────────────┘     └─────────────────┘
```

**功能模块**：
- **状态显示区**：实时显示处理任务统计
- **通知按钮**：显示未读通知数量
- **通知菜单**：包含通知列表、勿扰模式设置
- **设置面板**：通知类型过滤、勿扰模式配置

### 3. 通知管理系统架构

**通知类型**：
- 完成通知
- 错误/警告通知
- 系统状态通知
- 用户操作反馈

**状态管理**：
- 通知队列
- 已读/未读状态
- 勿扰模式
- 通知历史记录

## 实现计划

### 详细技术方案

## 会话状态指示器增强

**视觉设计**：
- **等待处理（灰色）**：`#6B7280` + 静态显示
- **正在处理（蓝色）**：`#3B82F6` + 呼吸动画
- **处理完成（绿色）**：`#10B981` + 淡出动画
- **处理错误（红色）**：`#EF4444` + 闪烁警告

### 状态指示器显示和消失逻辑

**状态生命周期管理**：

| 状态 | 显示条件 | 消失条件 | 显示时长 | 用户交互 |
|------|----------|----------|----------|----------|
| 等待处理（灰色） | 会话中有排队任务 | 所有等待任务开始处理或被取消 | 持续显示 | 无特殊交互 |
| 正在处理（蓝色） | 会话中有活跃处理任务 | 所有处理任务完成或失败 | 持续显示 | 无特殊交互 |
| 处理完成（绿色） | 任务成功完成 | 3秒后自动消失 | 3秒 | 可提前消失 |
| 处理错误（红色） | 任务处理失败 | 用户点击会话查看错误 | 持续显示 | 需要用户交互 |

**用户体验场景**：
- **批量处理**：多个文件依次处理，状态流畅转换
- **错误处理**：错误状态持续显示直到用户关注
- **性能优化**：批量状态更新，避免频繁UI刷新

**动画效果适配**：
- 暗色主题：增强对比度，确保可见性
- 亮色主题：适当降低饱和度，避免刺眼
- 高对比度模式：使用更明显的颜色差异

## 通知系统功能

**通知分类**：
- **任务通知**：完成、失败
- **系统通知**：应用更新、错误警告、维护提示
- **用户通知**：操作确认、结果反馈

**勿扰模式**：
- **全局勿扰**：屏蔽所有通知
- **分类勿扰**：只屏蔽特定类型通知
- **定时勿扰**：在指定时间段内勿扰

**通知设置**：
- 通知显示时长
- 通知声音开关
- 通知持久化（是否保存历史）
- 通知数量限制

## 扩展性考虑

- **插件化通知**：支持自定义通知类型
- **多语言支持**：通知内容国际化
- **通知历史**：可查看和搜索历史通知
- **导出功能**：支持通知日志导出
- **快捷键支持**：键盘快捷键操作通知
- **API接口**：第三方插件可集成通知系统

### 核心改进点

1. **会话标签指示器增强**
   - 从单一橙色指示器升级为多色状态系统
   - 蓝色（正在处理）、灰色（等待处理）、绿色（完成）、红色（错误）
   - 确保主题切换时的视觉一致性

2. **底部通知栏系统**
   - 左侧状态区域：实时显示任务统计
   - 右侧通知按钮：显示未读数量和快捷操作
   - 完整的通知菜单：消息列表、勿扰模式、通知设置

3. **通知管理系统**
   - 分类通知：任务、系统、用户通知
   - 灵活的勿扰模式：全局、分类、定时
   - 通知历史和设置持久化
