# 多线程改进计划（持续更新）

## 一、我判断的关键根因（基于当前代码结构）

### 根因 1：主线程仍在做重计算
在处理完成路径里，仍有视频缩略图生成、Shader 处理等重活落在主线程，拖动窗口时会抢占事件循环。

### 根因 2：任务启动阶段存在主线程阻塞
AI 任务启动时模型加载逻辑会触发较重的同步操作，容易在任务切换/并发启动时造成瞬时卡顿。

### 根因 3：UI 数据更新粒度过粗
当前存在多处“整表刷新式”更新（全量任务列表/消息状态同步），在处理中会反复触发 QML 重绘和绑定计算。

### 根因 4：会话同步写回频次偏高
处理过程中多次同步 session，会与 UI 事件竞争主线程时间片。

---

## 二、彻底解决方案（四层治理）

## Phase A：先做“可观测”基线（1 天）
目标：先量化，再优化，避免盲改。

1. 增加性能埋点（`QElapsedTimer`）：
   - `ProcessingController::startTask/completeTask/updateTaskProgress`
   - `MessageModel::updateProgress/updateStatus/updateFileStatus`
   - 会话同步入口
2. 建立卡顿指标：
   - 窗口拖动期间主线程长帧（>16.7ms, >33ms, >50ms）
   - 95/99 分位帧时长
3. 固定复现场景：
   - 10/30/50 文件处理并发 + 持续拖动窗口 30 秒

**产出**：一份“热点函数耗时 TopN + 卡顿帧分布”报告。

### A 阶段落地状态
- [x] `ProcessingController` 关键路径已加入耗时日志（`startTask/updateTaskProgress/completeTask/requestSessionSync`）
- [x] `MessageModel` 高频更新接口已加入耗时日志（`updateStatus/updateProgress`）
- [ ] 帧时长分位统计（95/99）仍需补齐（建议在 QML 帧回调或渲染统计中落地）

---

## Phase B：把重活彻底移出主线程（核心，2~3 天）

### B1. 处理完成链路异步化（高优先级）
把以下操作下沉到工作线程（可复用已有 `TaskCoordinator`/`QThreadPool`）：
- 视频缩略图生成
- Shader 预处理图生成
- 图像后处理（非 UI 相关）

主线程只做：
- 状态原子更新
- 最终结果信号派发
- 轻量模型通知

### B2. AI 模型加载策略优化（高优先级）
- 模型“预加载 + 复用缓存”
- 模型切换在工作线程执行
- 避免任务启动时在主线程做同步加载

### B3. 线程优先级与并发治理
- 工作线程优先级下调（确保 UI 线程抢占优先）
- 根据 CPU/GPU 压力动态调低并发上限（交互优先）

### B 阶段落地状态
- [x] 视频缩略图 + Shader 处理链路已异步化（`processShaderVideoThumbnailAsync`）
- [x] AI 模型已支持后台预加载（`requestModelPreload`）
- [x] 工作线程池优先级已下调（`QThread::LowPriority`）
- [x] 会话切换/批量导入场景资源让渡已实现（会话切换降级后台优先 + 导入期间并发削峰）

---

## Phase C：UI 更新机制重构（2 天）

### C1. 从“全量刷新”改为“增量刷新”
- 任务/消息模型改成按行、按 role 精准更新（`dataChanged`）
- 避免每次进度都触发整列表 `tasksChanged`

### C2. 统一更新节流与批处理
- 进度更新 66~100ms 批量合并派发
- 消息聚合进度按固定节奏刷新（例如 10~15Hz）
- 会话持久化改为 debounce（例如 300~500ms，或“任务阶段性完成”时写回）

### C3. QML 侧减负
- 列表 delegate 避免重计算绑定
- 缩略图统一 `sourceSize`，避免大图解码抖动
- 重型子组件改 `Loader` 延迟加载

### C 阶段落地状态
- [x] `ProcessingModel::tasksChanged` 已改为 66ms 节流合并
- [x] 会话列表 `notifySessionDataChanged` 已改为 120ms 批量通知
- [x] 处理进度同步与会话同步已加节流（80ms / 500ms）
- [x] 会话落盘从主线程同步写改为后台线程异步写（避免拖动窗口时写盘抖动）
- [ ] QML 重型 delegate 的 `Loader` 化与绑定瘦身仍需逐页排查

---

## Phase E（新增）：交互优先与会话切换隔离（1~2 天）
> 该阶段是“彻底解决拖动窗口、切换会话、重新上传媒体文件卡顿”的关键补强。

### E1. 会话切换期间的任务资源让渡
- 切换会话时，将旧会话活跃任务标记为后台优先级，并延后其高频 UI 回传。
- 目标：切换动作优先完成，避免“切换一下卡住半秒”。

### E2. 重新上传媒体文件时的瞬时削峰
- 当检测到批量导入/重传时，临时降低并发上限（如 `N -> max(1, N-1)`），导入完成后恢复。
- 避免 IO 解码、缩略图生成与处理中任务同时争抢主线程与磁盘。

### E3. 会话级写回隔离
- 活跃会话之外的消息状态更新仅更新内存，不立即触发 UI 对象链路。
- 将“会话内一致性”与“当前 UI 可见性”解耦，降低不可见会话对主界面的干扰。

### E 阶段落地状态
- [x] E1 已落地：会话切换时触发任务优先级让渡（旧会话降级 `Background`，新会话提升 `UserInteractive`）
- [x] E2 已落地：批量导入/重传自动进入并发削峰模式（`N -> max(1, N-1)`，队列清空后自动恢复）
- [ ] E3 待完善：目前已做到“仅活跃会话回传到 `MessageModel`”，仍需补齐“不可见会话的写回批处理窗口”

## Phase D：回归与验收（1 天）

### D1. 验收前准备（固定口径）
1. 使用 `Release` 构建并启动最新版本。
2. 清空或归档 `logs/runtime_output.log`，确保本轮日志可追溯。
3. 固定测试窗口尺寸（建议 1920x1080）与测试素材集（10/30/50 文件三组）。
4. 关闭无关后台任务（下载/杀毒全盘扫描等），减少噪声。

### D2. 三组压测场景（每组至少 30 秒）
- **场景 S1（10 文件）**：开始处理后持续拖动窗口 30 秒。
- **场景 S2（30 文件）**：处理中执行“拖动窗口 + 会话切换 + 返回”。
- **场景 S3（50 文件）**：处理中执行“拖动窗口 + 批量重传/导入 + 页面切换”。

每个场景都记录：
- 是否出现连续 >100ms 卡顿体感
- 是否出现明显掉帧/输入迟滞
- 任务状态、缩略图、会话恢复是否正确

### D3. 日志采集与统计口径
- 关键日志关键词：
  - `"[Perf][ProcessingController]"`
  - `"[Perf][MessageModel]"`
  - `"requestSessionSync debounce trigger"`
- 统计项：
  - `startTask/completeTask/updateTaskProgress` 慢路径次数
  - `updateProgress/updateStatus/updateFileStatus` 慢路径次数
  - `requestSessionSync` 触发频率（每分钟）

> 说明：当前帧 95/99 分位仍未接入渲染层统计，D 阶段先用“慢路径日志 + 体感 + 功能一致性”作为临时验收口径；后续补齐帧统计后再做一次严格验收。

### D4. 验收结论模板（建议直接贴到 `docs/Notes/`）
- 构建版本：`Release / 提交号`
- 测试日期：`YYYY-MM-DD`
- 场景结论：`S1/S2/S3 通过或不通过`
- 主要风险：`若有`
- 回归结论：`可合并 / 继续优化`

## 性能验收标准（必须全部通过）
1. 处理中拖动窗口：
   - 95% 帧时长 < 16.7ms（目标 60fps）
   - 99% 帧时长 < 24ms
   - 无连续 >100ms 卡顿
2. 功能一致性：
   - 任务状态流转（Pending/Processing/Completed/Failed）100%正确
   - 缩略图、结果路径、消息状态不丢失
   - 会话切换与恢复行为不回退
3. 压测：
   - 50 文件批处理 + 连续拖动 + 页面切换无明显顿挫

## 三、风险控制与不破坏现有功能策略

1. **分支隔离 + 分阶段提交**（每阶段可独立回滚）
2. **先加埋点，后改逻辑**，每步都有量化对比
3. **不改业务语义**（只改执行线程与更新节奏）
4. **关键路径加开关**（可临时切回旧路径进行对照）

---

## 四、建议执行顺序（更新版）

1. 先做 Phase A（确认热点耗时趋势）
2. 先吃满 B1 + B2 + C1 + C2（已落地大头）
3. 立刻补 E1 + E2（会话切换与重传时的交互优先）
4. 最后做 D 验收并固化文档
