## 报告安全漏洞

我们非常重视 EnhanceVision 的安全问题。如果您发现了安全漏洞，请**不要**通过公开的 GitHub Issues 报告。

### 报告方式

请通过以下方式私下报告安全漏洞：

1. **GitHub Security Advisories**（推荐）
   - 访问 [Security Advisories](https://github.com/K-irito02/EnhanceVision/security/advisories)
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **电子邮件**
   - 发送邮件至：saokiiritoasuna@qq.com
   - 邮件标题：`[Security] EnhanceVision Vulnerability Report`

### 报告内容

请在报告中包含以下信息：

- 漏洞类型（如：缓冲区溢出、XSS、权限提升等）
- 受影响的版本
- 复现步骤
- 概念验证代码（如适用）
- 漏洞影响评估
- 可能的修复建议（可选）

### 响应时间

| 阶段 | 预期时间 |
|------|----------|
| 确认收到报告 | 48 小时内 |
| 初步评估 | 7 天内 |
| 修复方案 | 视严重程度而定 |
| 发布补丁 | 修复完成后 7 天内 |

### 披露政策

- 请勿在补丁发布前公开披露漏洞
- 我们会在修复发布后公开致谢（如果您愿意）
- 对于严重漏洞，我们会发布安全公告

## 安全最佳实践

### 用户注意事项

1. **下载来源**
   - 仅从官方渠道下载 EnhanceVision
   - 验证下载文件的完整性

2. **AI 模型**
   - 仅使用来自可信来源的 AI 模型
   - 模型文件应放置在应用程序目录的 `models/` 文件夹中

3. **文件处理**
   - 处理来自不可信来源的文件时保持谨慎
   - 定期备份重要文件

### 开发者注意事项

1. **依赖更新**
   - 定期更新第三方依赖
   - 关注依赖的安全公告

2. **代码审查**
   - 所有代码变更需经过审查
   - 特别关注内存安全和输入验证

3. **敏感信息**
   - 不要在代码中硬编码敏感信息
   - 不要提交日志文件或配置文件中的敏感信息

## 已知安全限制

1. **本地处理**
   - EnhanceVision 完全在本地运行，不发送任何数据到外部服务器
   - 用户数据不会离开本地机器

2. **AI 模型**
   - AI 模型来自第三方项目，请确保从官方来源获取
   - 模型文件可能存在潜在风险，请谨慎处理
