# 消息展示区域优化实施计划

## 需求概述

对消息展示区域的消息卡片进行以下功能优化，确保用户体验流畅且符合预期行为：

1. **应用重启时消息定位优化**：重启后所有会话标签的最新消息自动滚动到底部
2. **区域覆盖规则**：根据最新消息卡片是否完整显示决定预览区域是否可覆盖
3. **消息发送后的自动定位**：发送完成后自动滚动到底部
4. **平滑滚动效果**：避免突兀的位置跳转

---

### 1.2 现有滚动逻辑

```qml
// MessageList.qml 现有属性

property bool autoScrollEnabled: true
property bool userHasScrolledUp: false

// 现有滚动方法
```

### 1.3 现有问题

1. **重启定位问题**：`Component.onCompleted` 中调用 `scrollToBottom()`，但会话切换时未重置滚动位置
2. **覆盖规则缺失**：未检测最新消息卡片可见性，预览区域和最小化区域的 z-index 固定
3. **发送后定位**：仅在 `onMessageAdded` 时滚动，未考虑发送完成场景
4. **滚动无动画**：直接调用 `positionViewAtEnd()` 无平滑过渡

## 二、实施方案

### 2.1 应用重启时消息定位优化

#### 2.1.1 实现思路

1. 在 `MessageList.qml` 中添加会话切换监听
2. 会话切换时重置 `userHasScrolledUp` 标志并滚动到底部
3. 应用启动时对所有会话执行初始化滚动

#### 2.1.2 修改文件

**MessageList.qml**：
- 添加 `sessionId` 属性，用于跟踪当前会话
- 添加 `Connections` 监听 `sessionController.activeSessionChanged`
- 在会话切换时调用 `_resetScrollState()`

```qml
// 新增属性
property string currentSessionId: ""

// 新增方法
function _resetScrollState() {
    userHasScrolledUp = false
    autoScrollEnabled = true
    Qt.callLater(scrollToBottomAnimated)
}

// 新增平滑滚动方法
function scrollToBottomAnimated() {
    if (messageList.count > 0) {
        scrollAnimation.to = messageList.contentHeight - messageList.height
        scrollAnimation.restart()
    }
}

// 新增滚动动画
NumberAnimation {
    id: scrollAnimation
    target: messageList
    property: "contentY"
    duration: 300
    easing.type: Easing.OutCubic
}
```

**MainPage.qml**：
- 传递 `currentSessionId` 给 MessageList
- 在 `onActiveSessionChanged` 中通知 MessageList 重置滚动状态

### 2.2 区域覆盖规则（简化版）

#### 2.2.1 实现思路

1. 在 `MessageList.qml` 中添加方法检测最新消息卡片是否完整显示
2. 直接根据可见性决定预览区域和最小化区域是否可覆盖最新消息
3. **无需跟踪用户滑动操作**

#### 2.2.2 覆盖规则

| 最新消息卡片状态 | 预览区/最小化区行为 |
|------------------|---------------------|
| 完整显示 | 不可覆盖（z-index 低于消息列表） |
| 不完整显示/不可见 | 可以覆盖（z-index 高于消息列表） |

#### 2.2.3 修改文件

**MessageList.qml**：
- 添加 `isLastMessageFullyVisible` 只读属性
- 添加 `canOverlayLastMessage` 只读属性（与 `isLastMessageFullyVisible` 相反）
- 暴露属性供外部查询覆盖权限

```qml
// 新增只读属性
readonly property bool isLastMessageFullyVisible: _checkLastMessageVisibility()
readonly property bool canOverlayLastMessage: !isLastMessageFullyVisible

// 新增方法
function _checkLastMessageVisibility() {
    if (messageList.count === 0) return true
    
    // 获取最后一个消息项
    var lastIndex = messageList.count - 1
    var lastDelegate = messageList.itemAt(lastIndex)
    if (!lastDelegate) return true
    
    // 计算最后一个消息项的可见区域
    var lastItemTop = lastDelegate.y
    var lastItemBottom = lastDelegate.y + lastDelegate.height
    var visibleTop = messageList.contentY
    var visibleBottom = messageList.contentY + messageList.height
    
    // 检查是否完整显示
    return lastItemTop >= visibleTop && lastItemBottom <= visibleBottom
}

// 监听内容变化更新可见性
onContentYChanged: _updateLastMessageVisibility()
onCountChanged: Qt.callLater(_updateLastMessageVisibility)
onHeightChanged: Qt.callLater(_updateLastMessageVisibility)

function _updateLastMessageVisibility() {
    // 触发属性重新计算
    isLastMessageFullyVisibleChanged()
}
```

**MainPage.qml**：
- 根据 `messageListView.canOverlayLastMessage` 动态调整预览区域和最小化区域的 z-index

```qml

```

### 2.3 消息发送后的自动定位

#### 2.3.1 实现思路

1. 监听 `processingController` 的任务完成信号
2. 在所有任务完成后（消息状态变为 Completed）自动滚动到底部
3. 仅在用户未手动滚动时执行

#### 2.3.2 修改文件

**MessageList.qml**：
- 添加 `Connections` 监听 `messageModel` 的消息状态变化
- 当消息状态从 Processing 变为 Completed 时，触发滚动

```qml
Connections {
    target: root._hasRealModel ? messageModel : null
    enabled: root._hasRealModel
    
    // 现有信号处理...
    
    // 新增：监听消息状态变化
    function onMessageStatusChanged(messageId, newStatus) {
        // 状态变为完成时，如果用户未手动滚动，则滚动到底部
        if (newStatus === 2 && !messageList.userHasScrolledUp) {
            Qt.callLater(messageList.scrollToBottomAnimated)
        }
    }
}
```

### 2.4 平滑滚动效果

#### 2.4.1 实现思路

1. 使用 `NumberAnimation` 替代直接设置 `contentY`
2. 添加滚动动画组件
3. 确保动画期间不响应用户交互

#### 2.4.2 修改文件

**MessageList.qml**：
- 添加平滑滚动动画组件
- 修改 `scrollToBottom()` 方法使用动画

```qml
// 平滑滚动动画
NumberAnimation {
    id: scrollAnimation
    target: messageList
    property: "contentY"
    duration: 300
    easing.type: Easing.OutCubic
    
    // 边界保护
    onRunningChanged: {
        if (!running) {
            // 确保最终位置正确
            messageList.positionViewAtEnd()
        }
    }
}

// 平滑滚动方法
function scrollToBottomAnimated() {
    if (messageList.count > 0) {
        var targetY = Math.max(0, messageList.contentHeight - messageList.height)
        if (Math.abs(messageList.contentY - targetY) > 5) {
            scrollAnimation.to = targetY
            scrollAnimation.restart()
        }
    }
}

// 强制滚动方法（无动画，用于初始化）
function scrollToBottomImmediate() {
    if (messageList.count > 0) {
        scrollAnimation.stop()
        messageList.positionViewAtEnd()
    }
}
```

## 四、风险评估与应对

### 4.1 潜在问题

| 问题 | 风险等级 | 应对方案 |
|------|----------|----------|
| 滚动动画与用户操作冲突 | 中 | 动画期间检测用户操作时停止动画 |
| z-index 调整导致点击穿透 | 低 | 确保 MouseArea 正确设置 |
| 会话切换时滚动状态丢失 | 低 | 在会话切换时显式重置状态 |
| 性能影响 | 低 | 使用 `Qt.callLater` 延迟执行，避免频繁计算 |

### 4.2 边界情况

1. **空消息列表**：所有方法需检查 `count > 0`
2. **快速会话切换**：使用 `Qt.callLater` 确保状态一致性
3. **大量消息**：滚动动画使用合理时长（300ms）
4. **消息项高度变化**：监听 `onContentHeightChanged` 更新可见性

## 五、验收标准

1. ✅ 应用重启后，所有会话标签的最新消息自动滚动到底部
2. ✅ 会话切换后，消息列表重置到最新消息位置
3. ✅ 最新消息卡片完整显示时，预览区/最小化区不可覆盖
4. ✅ 最新消息卡片不完整显示时，预览区/最小化区可以覆盖
5. ✅ 消息发送完成后，自动滚动到底部
6. ✅ 滚动过渡平滑，无突兀跳转
7. ✅ 不影响现有功能正常运行
