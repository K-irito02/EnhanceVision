# 消息卡片与放大查看器界面优化计划

## 一、任务概述

当主程序界面通过上下、左右拉伸和展开会话侧边栏与控制面板，使程序界面变得最小时，对消息卡片和放大查看器（嵌入式和独立式）进行界面优化，提升小屏幕下的用户体验。

## 二、优化目标

### 2.1 消息卡片优化

#### 2.1.1 左上角模式选择提示
**当前实现**：显示图标 + 文字（"Shader" 或 "AI"）
**优化目标**：只显示图标，不显示文字
**实现方案**：
- 在 `MessageItem.qml` 中，根据卡片宽度判断是否显示文字
- 当宽度小于某个阈值（如 350px）时，隐藏 `modeText` 文本

#### 2.1.2 左下角文件统计
**当前实现**：显示 "图标 成功/失败：x个文件"
**优化目标**：只显示图标＋数字，不显示文字
**实现方案**：
- 在 `MessageItem.qml` 中，根据卡片宽度判断显示模式
- 当宽度小于某个阈值时，简化显示为 "图标 + 数字"

#### 2.1.3 右上角功能按钮
**当前实现**：某些按钮在展开会话侧边栏时会被移动隐藏
**优化目标**：确保所有功能按钮图标完整显示
**实现方案**：
- 分析当前按钮的显示逻辑（`root.width >= 300` 等条件）
- 优化按钮布局，使用更紧凑或者自动向左再移动的排列方式，确保所有图标完整显示

#### 2.1.4 右下角文件统计与总耗时
**当前实现**：显示文件统计 + 总耗时
**优化目标**：不显示文件统计，只显示总耗时
**实现方案**：
- 在 `MessageItem.qml` 中，根据卡片宽度判断是否显示文件统计
- 当宽度小于某个阈值时，隐藏文件统计文本
- 保留总耗时显示

### 2.2 放大查看器优化

#### 2.2.1 创建新的 SVG 图标
**需要创建的图标**：
1. `playback-speed.svg` - 播放速度图标
2. `playback-settings.svg` - 播放设置图标
3. `volume-control.svg` - 音量控制图标（优化已有的 `volume.svg`图标）

**图标设计要求**：
- 简洁明了，符合项目整体风格
- 支持亮暗主题切换（通过 `ColoredIcon` 组件实现）
- 尺寸适中（建议 24x24 或 16x16）

#### 2.2.2 优化菜单样式
**当前实现**：菜单右侧存在多余的空白区域
**优化目标**：菜单上下左右四周与内容贴合，无多余空白
**实现方案**：
- 在 `MediaViewerControls.qml` 中，优化 `speedMenu` 和 `settingsMenu` 的样式
- 移除菜单的默认 padding 和 margin
- 使用自定义背景样式，确保紧凑布局
- 调整菜单项的间距和内边距

#### 2.2.3 优化音量条样式和方式
**当前实现**：点击音量图标后，弹出横向音量条
**优化目标**：竖着显示，只显示音量条，其它元素透明或删除
**实现方案**：
- 在 `MediaViewerControls.qml` 中，修改 `volumePopup` 的布局
- 将 Slider 改为竖直方向（`orientation: Qt.Vertical`）
- 调整 Popup 的尺寸，使其紧凑贴合音量条
- 移除或透明化背景元素，只保留音量条本身

#### 2.2.4 缩略图删除按钮
**当前实现**：删除按钮尺寸为 20x20
**优化目标**：缩小删除按钮尺寸
**实现方案**：
- 在 `MediaThumbnailDelegate.qml` 中，调整删除按钮的尺寸
- 建议缩小到 16x16 或 14x14
- 调整字体大小，确保视觉平衡

### 2.3 国际化支持

**目标**：所有悬停提示和显示文字支持中英文切换
**实现方案**：
- 使用 `qsTr()` 函数包裹所有用户可见的文本
- 更新翻译文件 `resources/i18n/app_zh_CN.ts`
- 确保新增的提示文本都有对应的翻译条目

### 2.4 主题支持

**目标**：图标支持亮暗主题切换
**实现方案**：
- 使用 `ColoredIcon` 组件，通过 `Theme.colors.icon` 等颜色属性动态着色
- 确保新增图标在不同主题下都有良好的可见性
- 测试亮暗主题切换时的图标显示效果

## 四、技术细节

### 4.1 响应式断点设计
根据 `ResponsiveUtils.qml` 的定义：
- `breakpointFull`: 700px - 完整模式
- `breakpointStandard`: 500px - 标准模式
- `compact`: <500px - 紧凑模式

建议为消息卡片添加类似的断点逻辑：
- 宽度 >= 400px：显示完整信息
- 宽度 300-400px：显示简化信息
- 宽度 < 300px：显示最简信息

### 4.2 图标颜色适配
使用 `ColoredIcon` 组件：
```qml
ColoredIcon {
    source: Theme.icon("icon-name")
    iconSize: 16
    color: Theme.colors.icon  // 自动适配主题
}
```

### 4.3 国际化文本格式
```qml
Text {
    text: qsTr("成功：%1个文件").arg(count)
}
```

## 五、风险与注意事项

1. **兼容性**：确保优化后的界面在不同分辨率下都能正常显示
2. **性能**：避免频繁的布局计算，使用条件渲染而非动态隐藏
3. **可访问性**：确保悬停提示功能正常，方便用户理解图标含义
4. **主题切换**：测试亮暗主题切换时的视觉效果
5. **国际化**：确保所有文本都有对应的翻译条目

## 六、预期成果

1. 消息卡片在小屏幕下显示更紧凑、更清晰
2. 放大查看器的控制栏更简洁、更易用
3. 所有文本支持中英文切换
4. 所有图标支持亮暗主题切换
5. 无警告、无错误的稳定运行状态
