# 视频自动播放功能优化计划

## 问题分析

### 问题1：按钮名称不够准确

当前名称：
- 打开自动播放
- 切换自动播放
- 恢复播放位置

**问题**：名称不够直观，用户难以理解功能含义。

### 问题2：设置页面缺少描述信息

当前设置页面的配置项只有开关，没有说明文字，用户不清楚功能作用。

### 问题3：视频控制栏按钮未显示

检查代码发现，"切换自动播放"和"恢复播放位置"两个按钮的代码已添加，但但可能由于布局或条件问题未正确显示。

### 问题4：设置开关功能不起作用

检查 SettingsPage.qml 的代码，发现 Connections 绑定的信号名称可能不正确，需要修复。

## 优化方案

### 1. 按钮和配置项重命名

| 功能 | 新名称 | 说明 |
|------|--------|------|
| 打开自动播放 | 开/切自动播放 | 视频进行放大查看（嵌入式和独立式）和点击左右导航按钮切换到视频时自动开始播放 |
| 切换自动播放 | 源/结自动播放 | 放大查看（嵌入式和独立式）切换源件/结果时自动播放 |
| 恢复播放位置 | 源/结恢复进度 | 放大查看（嵌入式和独立式）切换源件/结果时恢复播放进度 |
注：视频控制区域"开/切自动播放"，"源/结自动播放"，"源/结恢复进度"这三个按钮只显示svg图标（需要创建svg图标，确保符合三个按钮功能的图标），不显示说明文字。但是鼠标悬停在按钮上时，会显示对应的新名称。

### 2. 设置页面添加描述信息

为每个配置项添加描述文字：
- 开/切自动播放： "点击视频进行放大查看（嵌入式和独立式）和点击左右导航按钮切换到视频时自动开始播放"
- 源/结自动播放: "放大查看（嵌入式和独立式）切换源件/结果时自动播放"
- 源/结恢复进度: "放大查看（嵌入式和独立式）切换源件/结果时恢复播放进度"

### 3. 修复视频控制栏按钮显示问题

检查 MediaViewerWindow.qml 的按钮布局代码，确保三个按钮图标正确显示

### 4. 修复设置开关功能问题

检查 SettingsPage.qml 的 Connections 绑定，确保信号名称正确
