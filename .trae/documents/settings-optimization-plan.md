# 设置页面优化计划 - 数据路径与缓存管理

## 需求分析

### 1. 默认保存路径优化
**现状问题**：
- 当前只有"默认保存路径"（导出文件位置）支持自定义
- AI 处理结果、Shader 处理结果等应用数据固定存储在 C 盘 AppData 目录
- 用户无法将这些数据重定向到其他磁盘

**优化目标**：
- 支持用户自定义"应用数据目录"（存储处理结果等）
- 区分"必须存 C 盘"和"可自定义"的数据
- 提供友好的路径选择和迁移功能

### 2. 缓存管理优化
**现状问题**：
- 当前只有一个"清除缩略图缓存"按钮
- 缺少其他可清理数据的选项
- 没有显示各类数据的大小
- 缺少友好的提示和确认机制

**优化目标**：
- 列出所有可清理的数据类型
- 显示各类数据的大小
- 提供独立的删除按钮和批量清理功能
- 添加友好的提示描述和确认对话框

## 数据分类

### 必须存 C 盘（不可自定义/不可删除）
| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| 会话数据 | `AppDataLocation/sessions.json` | 用户会话记录，核心数据 |
| 应用设置 | `ConfigLocation/settings.ini` | 应用配置，核心数据 |
| AI 模型 | `应用目录/models/` | 应用资源，随程序分发 |

### 可自定义存储路径
| 数据类型 | 默认位置 | 说明 |
|---------|---------|------|
| 处理结果目录 | `AppLocalDataLocation/processed/` | AI/Shader 处理结果 |
| 默认导出路径 | `PicturesLocation/EnhanceVision/` | 用户导出文件位置 |

### 可清理数据
| 数据类型 | 存储位置 | 风险等级 | 说明 |
|---------|---------|---------|------|
| AI 处理结果 | `processed/ai/` | 低 | 可重新生成 |
| Shader 图像结果 | `processed/` | 低 | 可重新生成 |
| Shader 视频结果 | `processed/shader_video/` | 低 | 可重新生成 |
| 缩略图缓存 | 内存中 | 无 | 内存缓存，清理无影响 |
| 运行时日志 | `logs/runtime_output.log` | 无 | 每次启动覆盖 |
| 崩溃日志 | `logs/crash.log` | 低 | 用于问题排查 |

## 实施方案

### 第一阶段：C++ 后端扩展

#### 1.1 SettingsController 扩展
**新增属性**：
```cpp
// 自定义数据路径（空则使用默认路径）
Q_PROPERTY(QString customDataPath READ customDataPath WRITE setCustomDataPath NOTIFY customDataPathChanged)

// 数据大小信息（只读）
Q_PROPERTY(qint64 aiProcessedSize READ aiProcessedSize NOTIFY dataSizeChanged)
Q_PROPERTY(qint64 shaderImageSize READ shaderImageSize NOTIFY dataSizeChanged)
Q_PROPERTY(qint64 shaderVideoSize READ shaderVideoSize NOTIFY dataSizeChanged)
Q_PROPERTY(qint64 logSize READ logSize NOTIFY dataSizeChanged)
Q_PROPERTY(qint64 totalCacheSize READ totalCacheSize NOTIFY dataSizeChanged)
```

**新增方法**：
```cpp
Q_INVOKABLE void refreshDataSize();           // 刷新数据大小
Q_INVOKABLE bool clearAIProcessedData();      // 清理 AI 处理结果
Q_INVOKABLE bool clearShaderImageData();      // 清理 Shader 图像结果
Q_INVOKABLE bool clearShaderVideoData();      // 清理 Shader 视频结果
Q_INVOKABLE bool clearLogs();                 // 清理日志
Q_INVOKABLE bool clearAllCache();             // 清理所有缓存
Q_INVOKABLE QString formatSize(qint64 bytes); // 格式化大小显示
```

#### 1.2 ProcessingController 修改
- 修改输出路径逻辑，支持自定义数据路径
- 当 `customDataPath` 不为空时，使用自定义路径

#### 1.3 ThumbnailProvider 扩展
```cpp
Q_INVOKABLE void clearCache();  // 清理缩略图缓存
Q_INVOKABLE int cacheCount();   // 缓存数量
```

## 风险与注意事项

1. **路径迁移**：用户更改数据目录后，需要考虑是否迁移现有数据
   - 建议：仅更改新数据的存储位置，不自动迁移旧数据
   - 可选：提供"迁移现有数据"选项

2. **路径验证**：需要验证用户选择的路径是否有效
   - 检查写入权限
   - 检查磁盘空间
   - 处理路径不存在的情况

3. **清理确认**：所有清理操作都应有确认对话框
   - 防止误操作
   - 清晰说明清理后果

4. **性能考虑**：计算目录大小可能耗时
   - 使用异步计算
   - 缓存计算结果
   - 提供刷新按钮
