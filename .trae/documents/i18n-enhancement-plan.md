# 国际化完善实施计划

## 任务概述

完善 EnhanceVision 应用的国际化支持，确保所有界面文本实现中英双语翻译，并根据系统语言设置自动切换显示语言。

## 问题分析

### 1. 自动重新处理警告弹窗翻译问题

**位置**: `qml/App.qml` 第274-322行

**当前状态**:
- 弹窗文本已使用 `qsTr()` 包裹
- 翻译文件中已有对应条目（在 SettingsPage context 中）

**需要修复**:
- 翻译条目分散在不同 context 中，需要统一管理
- 需要确保弹窗文本在 App context 中有独立翻译条目

### 2. 新建会话标签语言适配问题

**位置**:
- `src/controllers/SessionController.cpp` 第554行
- `src/models/SessionModel.cpp` 第337行

**当前状态**:
- 使用 `tr("未命名会话 %1")` 生成默认会话名
- 中文翻译文件中翻译为空

**需要修复**:
- 补充中文翻译文件中的翻译
- 确保英文翻译文件中有 "Untitled Session %1" 翻译

### 3. 界面文本翻译缺失问题

**发现的问题**:

#### 3.1 中文翻译文件 `app_zh_CN.ts` 中存在大量空翻译
约100处 `<translation ></translation>` 需要补充中文翻译

#### 3.2 部分文本未使用翻译函数
以下文本直接硬编码，需要评估是否需要翻译：
- `"E"` - 品牌标识，可能不需要翻译
- `"EnhanceVision"` - 品牌名称，不需要翻译
- `"v0.1"`, `"v0.1.0"` - 版本号，不需要翻译
- `"Built with Qt6 + NCNN"` - 技术说明，需要翻译
- `"Shader"` - 技术术语，需要翻译

#### 3.3 特殊符号（不需要翻译）
- `"×"` (关闭按钮)
- `"−"`, `"+"` (调节按钮)
- `"•"` (列表标记)
- `"中"`, `"En"` (语言切换指示器)

## 风险评估

1. **翻译一致性**: 需确保同一术语在不同位置翻译一致
2. **文本长度**: 英文文本可能比中文长，需检查UI布局
3. **特殊字符**: 引号等特殊字符在翻译文件中需正确转义
