# 主题切换问题修复计划

## 问题概述

根据 `docs/Plan/主题切换功能技术分析报告.md` 中指出的问题，需要解决以下三类问题：

### 问题 1: 状态同步冗余
- **位置**: `qml/pages/SettingsPage.qml:110`, `qml/pages/SettingsPage.qml:127`
- **原因**: `Theme.setDark()` 内部已经调用了 `SettingsController.theme`，但组件又重复调用
- **影响**: 代码冗余，可能导致状态不一致

### 问题 2: 双图标集维护成本高
- **位置**: `resources/icons/` (68个文件) 和 `resources/icons-dark/` (68个文件)
- **原因**: 维护两套图标资源，切换时触发图片重加载导致闪烁
- **影响**: 包体积冗余、维护成本高、切换闪烁

### 问题 3: 硬编码颜色分散
- **位置**: 多个 QML 文件中存在硬编码颜色值
- **统计**: 252 处硬编码颜色（排除 Colors.qml 后仍有约 100 处）
- **影响**: 主题切换时颜色不一致，可维护性差

---

## 实施步骤

### 步骤 1: 修复状态同步冗余

**文件**: `qml/pages/SettingsPage.qml`

**修改内容**:
```qml
// 修改前 (第110行)
MouseArea { anchors.fill: parent; cursorShape: Qt.PointingHandCursor; onClicked: { Theme.setDark(true); SettingsController.theme = "dark" } }

// 修改后
MouseArea { anchors.fill: parent; cursorShape: Qt.PointingHandCursor; onClicked: { Theme.setDark(true) } }

// 修改前 (第127行)
MouseArea { anchors.fill: parent; cursorShape: Qt.PointingHandCursor; onClicked: { Theme.setDark(false); SettingsController.theme = "light" } }

// 修改后
MouseArea { anchors.fill: parent; cursorShape: Qt.PointingHandCursor; onClicked: { Theme.setDark(false) } }
```

---

### 步骤 2: 删除双图标集

**操作**:
1. 删除 `resources/icons-dark/` 目录及其所有文件
2. 从 `CMakeLists.txt` 中移除所有 `resources/icons-dark/` 相关的引用
3. 从 `resources/qml.qrc` 中移除所有 `icons-dark` 相关的引用

**CMakeLists.txt 需要删除的行** (约 60 行):
- 第 361-428 行: 所有 `resources/icons-dark/` 开头的资源引用

**qml.qrc 需要删除的行**:
- 所有 `<file>icons-dark/...</file>` 行

---

### 步骤 3: 修复硬编码颜色

**需要修复的文件**:

| 文件 | 硬编码数量 | 典型问题 |
|------|-----------|----------|
| `ShaderParamsPanel.qml` | 2 | `#1E1E2E`, `#FFFFFF` |
| 其他组件文件 | 若干 | 需逐个检查 |

**修复方案**:
- `#FFFFFF` → `Theme.colors.card` 或 `Theme.colors.foreground` (根据上下文)
- `#1E1E2E` → `Theme.colors.card` (暗色主题卡片背景)
- 其他硬编码颜色根据语义替换为对应的 `Theme.colors.*` 属性
