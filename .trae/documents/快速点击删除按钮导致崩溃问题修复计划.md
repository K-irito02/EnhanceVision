# 快速点击删除按钮导致崩溃问题修复计划

## 问题描述

在消息卡片中（正在处理的任务、待处理的任务、已完成的任务、失败的任务），当高压连续快速点击缩略图右上角的删除按钮时，出现以下情况：
1. **Shader模式**：闪退
2. **AI推理模式**：界面变成无响应状态，最后崩溃

## 问题分析

### 1. 问题根源

通过代码分析，发现问题主要出现在以下几个层面：

#### 1.1 QML 层面的防抖机制不足

**文件**: `qml/components/MediaThumbnailStrip.qml`

```javascript
// 第645-652行
onClicked: {
    if (thumbDelegate._deleteInProgress) return
    thumbDelegate._deleteInProgress = true
    thumbDelegate._deleteOpacity = 0.3
    debounceTimer.start()  // 200ms
    var origIndex = thumbDelegate.itemData ? thumbDelegate.itemData.origIndex : thumbDelegate.index
    root.deleteFile(origIndex)
}
```

**问题**：
- 防抖时间太短（200ms），高压快速点击可能穿透
- 防抖标志 `_deleteInProgress` 是每个缩略图实例独立的，不同缩略图的点击不会被防抖
- 没有全局的删除操作锁，多个删除操作可能同时进行

#### 1.2 MessageItem 层面的防抖机制不足

**文件**: `qml/components/MessageItem.qml`

```javascript
// 第380-398行
onDeleteFile: function(index) {
    if (root._messageDeleteLocked) return
    root._messageDeleteLocked = true
    root._deletingOverlayOpacity = 1
    messageDeleteDebounceTimer.start()  // 300ms
    var _msgId = root.taskId
    var _fileId = ""
    if (root.mediaFiles && index >= 0 && index < root.mediaFiles.count) {
        var _item = root.mediaFiles.get(index)
        if (_item) _fileId = _item.id
    }
    var __msgId = _msgId
    var __fileId = _fileId
    Qt.callLater(function() {
        if (__fileId) {
            messageModel.removeMediaFileById(__msgId, __fileId)
        }
    })
}
```

**问题**：
- 防抖时间太短（300ms）
- 防抖标志 `_messageDeleteLocked` 是每个消息实例独立的，不同消息的删除不会被防抖
- 使用 `Qt.callLater` 延迟执行，但延迟时间不可控

#### 1.3 C++ 层面的并发控制不足

**文件**: `src/models/MessageModel.cpp`

```cpp
// 第816-883行
bool MessageModel::removeMediaFile(const QString &messageId, int fileIndex)
{
    // ...
    if (m_removingMessageIds.contains(messageId)) {
        return true;  // 如果正在删除，直接返回
    }
    m_removingMessageIds.insert(messageId);  // 标记为正在删除

    QTimer::singleShot(0, this, [this, messageId, removedFileId]() {
        // 异步删除逻辑
        if (m_processingController && !removedFileId.isEmpty()) {
            m_processingController->cancelMessageFileTasks(messageId, removedFileId);
        }
        // ... 删除文件和更新模型
        m_removingMessageIds.remove(messageId);  // 删除完成后移除标记
    });

    return true;
}
```

**问题**：
- `m_removingMessageIds` 是消息级别的防抖，不同消息的删除可以同时进行
- 使用 `QTimer::singleShot(0, ...)` 异步执行，可能导致竞态条件
- 如果删除操作失败或卡住，`m_removingMessageIds` 可能不会被清除

#### 1.4 ProcessingController 层面的取消逻辑问题

**文件**: `src/controllers/ProcessingController.cpp`

```cpp
// 第1819-1828行
void ProcessingController::cancelVideoProcessing(const QString& taskId)
{
    if (m_activeVideoProcessors.contains(taskId)) {
        auto processor = m_activeVideoProcessors[taskId];
        if (processor) {
            processor->cancel();  // 只是设置取消标志，不等待完成
        }
        m_activeVideoProcessors.remove(taskId);  // 立即移除
    }
}
```

**问题**：
- `cancel()` 只是设置取消标志，不等待处理完成
- 立即从 `m_activeVideoProcessors` 中移除，可能导致：
  - 处理器还在运行，但已经被移除
  - 后续的完成信号找不到对应的处理器
  - 资源访问冲突

#### 1.5 VideoProcessor 和 AIVideoProcessor 的取消逻辑问题

**VideoProcessor（Shader模式）**：
- `cancel()` 只是设置 `m_cancelled = true`
- 在处理过程中检查 `m_cancelled`，如果为 true 则抛出异常
- 没有等待处理完成的机制

**AIVideoProcessor（AI推理模式）**：
- `cancel()` 只是设置 `m_impl->cancelled.store(true)`
- `waitForFinished()` 会阻塞主线程（如果在主线程调用）
- 如果处理时间较长，会导致界面无响应

### 2. 崩溃原因推断

#### 2.1 Shader模式闪退

1. 快速连续点击删除按钮
2. 多个删除操作同时进行
3. `cancelVideoProcessing()` 被多次调用
4. `VideoProcessor` 被取消但仍在运行
5. FFmpeg 资源被提前释放
6. 处理线程访问已释放的资源 → **崩溃**

#### 2.2 AI推理模式界面无响应

1. 快速连续点击删除按钮
2. 多个删除操作同时进行
3. `cancelMessageFileTasks()` 被调用
4. `terminateTask()` 被调用
5. `waitForFinished()` 阻塞主线程
6. 处理时间较长 → **界面无响应**
7. 超时后资源访问冲突 → **崩溃**

## 修复方案

### 方案概述

采用多层防护策略，从 QML 到 C++ 层层加锁，确保删除操作的原子性和互斥性。

### 修复步骤

#### 步骤1：QML 层面 - 增强防抖机制

**文件**: `qml/components/MediaThumbnailStrip.qml`

**修改内容**：
1. 增加防抖时间到 500ms
2. 添加全局删除锁（通过 root 属性传递）
3. 在删除操作开始时禁用删除按钮

**具体修改**：
```javascript
// 在 delegate 中
property bool _deleteInProgress: false
property real _deleteOpacity: 1.0

Timer {
    id: debounceTimer
    interval: 500  // 增加到 500ms
    repeat: false
    onTriggered: {
        thumbDelegate._deleteInProgress = false
        thumbDelegate._deleteOpacity = 1.0
    }
}

// 删除按钮点击处理
onClicked: {
    if (thumbDelegate._deleteInProgress) return
    if (root.deleteLocked) return  // 检查全局锁
    
    thumbDelegate._deleteInProgress = true
    thumbDelegate._deleteOpacity = 0.3
    debounceTimer.start()
    
    var origIndex = thumbDelegate.itemData ? thumbDelegate.itemData.origIndex : thumbDelegate.index
    root.deleteFile(origIndex)
}
```

**文件**: `qml/components/MessageItem.qml`

**修改内容**：
1. 增加防抖时间到 500ms
2. 添加全局删除锁（通过全局单例或信号传递）
3. 在删除操作开始时显示加载状态

#### 步骤3：MessageModel 层面 - 增强并发控制

**文件**: `src/models/MessageModel.cpp`

**修改内容**：
1. 添加全局删除锁（跨消息）
2. 增加删除操作的超时保护
3. 确保删除操作的原子性

**具体修改**：
```cpp
// 在 MessageModel.h 中添加
QSet<QString> m_removingMessageIds;
QSet<QString> m_removingFileIds;  // 新增：文件级别的防抖
QTimer* m_removeCleanupTimer = nullptr;  // 新增：超时清理定时器

// 在 MessageModel.cpp 中修改
bool MessageModel::removeMediaFile(const QString &messageId, int fileIndex)
{
    int msgIdx = findMessageIndex(messageId);
    if (msgIdx < 0) {
        emit errorOccurred(tr("消息不存在: %1").arg(messageId));
        return false;
    }

    Message &message = m_messages[msgIdx];
    if (fileIndex < 0 || fileIndex >= message.mediaFiles.size()) {
        emit errorOccurred(tr("文件索引无效: %1").arg(fileIndex));
        return false;
    }

    const QString removedFileId = message.mediaFiles[fileIndex].id;
    
    // 文件级别的防抖
    if (m_removingFileIds.contains(removedFileId)) {
        return true;
    }
    m_removingFileIds.insert(removedFileId);
    
    // 消息级别的防抖
    if (m_removingMessageIds.contains(messageId)) {
        return true;
    }
    m_removingMessageIds.insert(messageId);

    // 设置超时清理（5秒）
    if (!m_removeCleanupTimer) {
        m_removeCleanupTimer = new QTimer(this);
        m_removeCleanupTimer->setSingleShot(true);
        connect(m_removeCleanupTimer, &QTimer::timeout, this, [this]() {
            qWarning() << "[MessageModel] Remove operation timeout, clearing locks";
            m_removingMessageIds.clear();
            m_removingFileIds.clear();
        });
    }
    m_removeCleanupTimer->start(5000);

    QTimer::singleShot(0, this, [this, messageId, removedFileId]() {
        // ... 删除逻辑
        
        // 清理防抖标志
        m_removingMessageIds.remove(messageId);
        m_removingFileIds.remove(removedFileId);
        if (m_removeCleanupTimer) {
            m_removeCleanupTimer->stop();
        }
    });

    return true;
}
```

#### 步骤4：ProcessingController 层面 - 改进取消逻辑

**文件**: `src/controllers/ProcessingController.cpp`

**修改内容**：
1. 添加取消操作的全局锁
2. 改进 `cancelVideoProcessing()` 的逻辑，确保异步取消
3. 添加取消操作的超时保护

**具体修改**：
```cpp
// 在 ProcessingController.h 中添加
QSet<QString> m_cancellingTaskIds;  // 正在取消的任务
QMutex m_cancelMutex;  // 取消操作的互斥锁

// 在 ProcessingController.cpp 中修改
void ProcessingController::cancelVideoProcessing(const QString& taskId)
{
    QMutexLocker locker(&m_cancelMutex);
    
    if (m_cancellingTaskIds.contains(taskId)) {
        return;  // 已经在取消中
    }
    m_cancellingTaskIds.insert(taskId);
    
    if (m_activeVideoProcessors.contains(taskId)) {
        auto processor = m_activeVideoProcessors[taskId];
        if (processor) {
            processor->cancel();  // 设置取消标志
            
            // 异步等待取消完成
            QTimer::singleShot(0, this, [this, taskId, processor]() {
                // 等待取消完成（最多 2 秒）
                QElapsedTimer timer;
                timer.start();
                while (processor->isProcessing() && timer.elapsed() < 2000) {
                    QThread::msleep(10);
                    QCoreApplication::processEvents();
                }
                
                // 清理
                m_activeVideoProcessors.remove(taskId);
                m_cancellingTaskIds.remove(taskId);
            });
        }
    } else {
        m_cancellingTaskIds.remove(taskId);
    }
}
```

**具体修改（AIVideoProcessor）**：
```cpp
// 在 AIVideoProcessor.h 中添加
signals:
    void cancelled();

// 在 AIVideoProcessor.cpp 中修改
void AIVideoProcessor::cancel()
{
    if (!m_impl->cancelled.load()) {
        qInfo() << "[AIVideoProcessor] Cancel requested";
        m_impl->cancelled.store(true);
        emit cancelled();
    }
}

// 改进 waitForFinished，避免阻塞主线程
void AIVideoProcessor::waitForFinished(int timeoutMs)
{
    if (m_processingFuture.isRunning()) {
        qInfo() << "[AIVideoProcessor] Waiting for processing to finish, timeout:" << timeoutMs << "ms";
        
        // 使用事件循环等待，避免阻塞
        QElapsedTimer timer;
        timer.start();
        
        while (m_processingFuture.isRunning() && timer.elapsed() < timeoutMs) {
            QCoreApplication::processEvents(QEventLoop::ExcludeUserInputEvents);
            QThread::msleep(10);
        }
        
        if (m_processingFuture.isRunning()) {
            qWarning() << "[AIVideoProcessor] Wait timeout, processing still running";
        } else {
            qInfo() << "[AIVideoProcessor] Processing finished after" << timer.elapsed() << "ms";
        }
    }
}
```

#### 步骤6：添加全局删除锁管理器

**新文件**: `src/utils/DeleteLockManager.h` 和 `src/utils/DeleteLockManager.cpp`

**目的**：统一管理所有删除操作的锁，确保全局互斥

**具体实现**：
```cpp
// DeleteLockManager.h
#pragma once

#include <QObject>
#include <QSet>
#include <QMutex>
#include <QTimer>

namespace EnhanceVision {

class DeleteLockManager : public QObject
{
    Q_OBJECT

public:
    static DeleteLockManager* instance();

    bool tryLock(const QString& key);
    void unlock(const QString& key);
    bool isLocked(const QString& key) const;
    void clearAllLocks();

signals:
    void lockAcquired(const QString& key);
    void lockReleased(const QString& key);

private:
    explicit DeleteLockManager(QObject* parent = nullptr);

    mutable QMutex m_mutex;
    QSet<QString> m_lockedKeys;
    QTimer* m_cleanupTimer = nullptr;
};

} // namespace EnhanceVision

// DeleteLockManager.cpp
#include "DeleteLockManager.h"
#include <QCoreApplication>

namespace EnhanceVision {

DeleteLockManager* DeleteLockManager::instance()
{
    static DeleteLockManager* instance = new DeleteLockManager(QCoreApplication::instance());
    return instance;
}

DeleteLockManager::DeleteLockManager(QObject* parent)
    : QObject(parent)
{
    // 设置定期清理（防止死锁）
    m_cleanupTimer = new QTimer(this);
    m_cleanupTimer->setInterval(10000);  // 10秒清理一次
    connect(m_cleanupTimer, &QTimer::timeout, this, [this]() {
        QMutexLocker locker(&m_mutex);
        if (!m_lockedKeys.isEmpty()) {
            qWarning() << "[DeleteLockManager] Clearing stale locks:" << m_lockedKeys.size();
            m_lockedKeys.clear();
        }
    });
    m_cleanupTimer->start();
}

bool DeleteLockManager::tryLock(const QString& key)
{
    QMutexLocker locker(&m_mutex);
    if (m_lockedKeys.contains(key)) {
        return false;
    }
    m_lockedKeys.insert(key);
    emit lockAcquired(key);
    return true;
}

void DeleteLockManager::unlock(const QString& key)
{
    QMutexLocker locker(&m_mutex);
    m_lockedKeys.remove(key);
    emit lockReleased(key);
}

bool DeleteLockManager::isLocked(const QString& key) const
{
    QMutexLocker locker(&m_mutex);
    return m_lockedKeys.contains(key);
}

void DeleteLockManager::clearAllLocks()
{
    QMutexLocker locker(&m_mutex);
    m_lockedKeys.clear();
}

} // namespace EnhanceVision
```

### 修复优先级

1. **高优先级**：步骤3、步骤4、步骤5（C++ 层面的并发控制）
2. **中优先级**：步骤6（全局删除锁管理器）
3. **低优先级**：步骤1、步骤2（QML 层面的防抖）
