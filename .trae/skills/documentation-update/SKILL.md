# 文档更新工作流

完成功能开发或问题修复后，系统化更新项目文档的工作流程。

## 核心原则

1. **及时更新** - 完成变更后立即更新文档
2. **保持一致** - 文档内容与代码实现同步
3. **简洁明了** - 避免冗余，突出关键信息

## 触发条件

| 场景 | 优先级 | 必须更新的文档 |
|------|--------|----------------|
| 新功能开发 | 高 | 笔记、README、规则文件 |
| Bug 修复 | 中 | 笔记、变更日志 |
| 代码重构 | 中 | 笔记、规则文件 |
| 新增组件/模块 | 高 | 笔记、README |
| 架构变更 | 高 | 笔记、规则文件、README |

## 执行流程

### 第一步：创建/更新开发笔记

在 `docs/Notes/` 目录下创建或更新笔记文件。

#### 文件命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 功能开发 | `<feature>-implementation-notes.md` | `dark-theme-implementation-notes.md` |
| Bug 修复 | `<issue>-fix-notes.md` | `memory-leak-fix-notes.md` |
| 重构 | `<refactor>-notes.md` | `session-model-refactor-notes.md` |

#### 笔记模板

```markdown
# <标题>

## 概述

<功能或问题概述>

**创建日期**: YYYY-MM-DD
**相关 Issue**: #xxx（如有）

---

## 变更概述

### 新增文件

| 文件路径 | 功能描述 |
|----------|----------|
| `path/to/file.cpp` | 功能说明 |

### 修改文件

| 文件路径 | 修改内容 |
|----------|----------|
| `path/to/file.cpp` | 修改说明 |

---

## 实现的功能特性

- ✅ 功能点 1：详细描述
- ✅ 功能点 2：详细描述

---

## 技术实现细节

### 关键代码片段

```cpp

```

### 设计决策

1. **决策 1**：原因和考量

---

## 遇到的问题及解决方案

### 问题 1：<问题描述>

**现象**：...

**原因**：...

**解决方案**：...

---

## 测试验证

| 场景 | 预期结果 | 实际结果 |
|------|----------|----------|
| 场景 1 | ... | ✅ 通过 |

---

## 后续工作

- [ ] 待优化项
```

### 第二步：更新变更日志

检查并更新 `CHANGELOG.md` 或 `docs/CHANGELOG.md`。

```markdown
## [Unreleased]

### Added
- 新增功能描述

### Changed
- 变更描述

### Fixed
- 修复描述

## [1.0.0] - 2024-01-15

### Added
- 初始版本功能列表
```

### 第三步：检查规则文件

检查 `.trae/rules/` 目录下的规则文件是否需要更新。

| 规则文件 | 更新条件 |
|----------|----------|
| `01-project-overview.md` | 新增技术或目录 |
| `02-architecture.md` | 架构变更 |
| `05-build-system.md` | 构建变更 |
| `10-skills-and-tools.md` | 新增技能 |

### 第四步：更新 README 文件

| 检查项 | 说明 |
|--------|------|
| 项目概述 | 是否需要更新功能描述 |
| 技术栈 | 是否有新增技术 |
| 目录结构 | 是否有新增目录 |
| 安装说明 | 是否有新的依赖或步骤 |

### 第五步：更新国际化文件（如适用）

```powershell
# 更新翻译源文件
lupdate CMakeLists.txt -ts resources/i18n/app_zh_CN.ts

# 使用 Qt Linguist 编辑翻译
linguist resources/i18n/app_zh_CN.ts

# 生成翻译二进制文件
lrelease resources/i18n/app_zh_CN.ts
```

## 输出清单

### 必须完成

- [ ] 开发笔记已创建/更新
- [ ] 变更日志已更新（如有重大变更）
- [ ] 规则文件已检查并更新（如需要）

### 按需完成

- [ ] README 文件已更新（如有结构/功能变更）
- [ ] 国际化文件已更新（如有文本变更）

## 文档质量检查

- [ ] 所有代码示例可运行
- [ ] 所有链接有效
- [ ] 所有路径正确
- [ ] Markdown 语法正确
