# 性能优化规则

## 线程模型

| 线程 | 职责 |
|------|------|
| 主线程 | UI 渲染与轻量逻辑 |
| 工作线程 | IO、图像处理、AI 推理、视频处理 |

- 跨线程通过信号槽（队列连接）通信
- 推理前在主线程快照模型与参数，工作线程只持快照

## 并发调度系统

| 组件 | 配置 |
|------|------|
| 任务优先级 | Critical/High/Normal/Low 四级 |
| AI引擎池 | 2 个并发 AI 推理任务 |
| 死锁检测 | 10 秒检查间隔 |
| 超时监控 | 5 秒检查间隔，默认 5 分钟超时 |
| 重试策略 | 指数退避，最大 3 次 |
| 饥饿防护 | 30 秒等待后自动提升优先级 |

## QML 渲染

- ✅ 列表优先 `ListView`，避免大规模 `Repeater`
- ✅ 重型组件使用 `Loader` 延迟加载
- ✅ 图像设置 `sourceSize`，避免原图无控制解码

## 动画与布局

- ✅ 优先动画 `opacity/transform`
- ❌ 避免高频动画 `width/height/x/y`
- ✅ 控制同屏持续动画数量

## GPU 内存管理

- AI推理前检查GPU可用内存，动态调整分块大小
- OOM时自动重试更小分块（128→96→64）
- 模型切换前清理 Vulkan allocator
- 大模型使用镜像填充边界（padding: 64/48/24px）

## 性能基线

| 指标 | 要求 |
|------|------|
| 关键交互 | 无明显卡顿 |
| 列表滚动 | 保持流畅 |
| AI推理OOM | 自动恢复，不崩溃 |
