# 项目技能和 MCP 服务指南

## 概述

本文档说明当前项目可用的技能和 MCP 服务，以及它们的使用场景和最佳实践。

## 可用技能 (Skills)

### 1. qt-build-and-fix
**作用**: 自动化构建、运行和修复 EnhanceVision 项目
- **使用场景**: 当需要编译项目、运行测试、检查代码质量或解决编译问题时
- **触发方式**: 用户要求构建项目、运行程序、修复编译错误等
- **功能**: 
  - CMake 项目构建
  - 程序运行和测试
  - 编译错误自动修复
  - 代码质量检查

### 2. qt-architecture
**作用**: Qt 应用程序架构设计和项目结构规划
- **使用场景**: 设计 Qt 应用架构、组织项目结构、选择设计模式
- **功能**:
  - MVC/MVP 模式选择
  - 项目目录结构规划
  - QApplication 设置
  - 组件分离和依赖管理

### 3. qt-layouts
**作用**: Qt 布局管理和组件排列
- **使用场景**: 布局管理、控件排列、响应式设计
- **功能**:
  - 布局管理器使用
  - 控件尺寸和位置控制
  - 响应式布局设计
  - 布局问题调试

### 4. qt-debugging
**作用**: Qt 应用程序调试和问题诊断
- **使用场景**: 程序崩溃、性能问题、逻辑错误调试
- **功能**:
  - 调试技术指导
  - 日志分析
  - 内存泄漏检测
  - 性能优化建议

### 5. qt-qml
**作用**: QML 和 Qt Quick 开发指导
- **使用场景**: QML 组件开发、属性绑定、信号处理
- **功能**:
  - QML 最佳实践
  - 组件设计模式
  - 属性绑定和信号槽
  - C++ 与 QML 交互

### 6. frontend-design
**作用**: 现代前端界面设计和开发
- **使用场景**: UI/UX 设计、界面美化、用户体验优化
- **功能**:
  - 现代设计系统应用
  - 响应式设计
  - 交互设计
  - 视觉效果实现

### 7. fullstack-developer
**作用**: 全栈 Web 开发
- **使用场景**: Web 应用开发、API 设计、数据库集成
- **功能**:
  - 前后端开发
  - 数据库设计
  - API 开发
  - 部署配置

### 8. code-reviewer
**作用**: 代码审查和质量保证
- **使用场景**: 代码提交前审查、PR 检查、代码质量评估
- **功能**:
  - 代码规范检查
  - 性能分析
  - 安全性审查
  - 最佳实践建议

### 9. documentation-update
**作用**: 文档更新和维护
- **使用场景**: 功能开发完成后更新文档、API 文档、用户手册
- **功能**:
  - 技术文档编写
  - API 文档生成
  - 用户指南更新
  - 文档结构优化

### 10. update-docs
**作用**: Next.js 文档更新（专门用于 Web 项目）
- **使用场景**: Next.js 项目文档更新、MDX 文件处理
- **功能**:
  - Next.js 文档同步
  - MDX 文件处理
  - 组件文档更新

## 可用 MCP 服务

### 1. Tavily MCP Server
**服务**: `mcp0_tavily-search`, `mcp0_tavily-extract`
- **作用**: 网络搜索和内容提取
- **使用场景**:
  - 搜索技术文档和解决方案
  - 提取网页内容用于参考
  - 获取最新的技术信息
- **示例**: 搜索 Qt 问题解决方案、提取文档内容

### 2. Context7 MCP Server
**服务**: `mcp1_resolve-library-id`, `mcp1_query-docs`
- **作用**: 获取最新的库文档和代码示例
- **使用场景**:
  - 查询 Qt、QML、JavaScript 等库的文档
  - 获取代码示例和最佳实践
  - 解决 API 使用问题
- **示例**: 查询 Qt 组件用法、获取 QML 属性说明

### 3. Playwright MCP Server
**服务**: `mcp4_*` 系列
- **作用**: Web 应用自动化测试和浏览器控制
- **使用场景**:
  - Web 应用功能测试
  - 浏览器自动化操作
  - UI 测试和截图
  - 性能测试
- **示例**: 测试 Web 界面、自动化浏览器操作

### 4. Postman API MCP Server
**服务**: `mcp5_*` 系列
- **作用**: API 测试和管理
- **使用场景**:
  - API 测试和调试
  - 接口文档管理
  - API 性能分析
  - 团队 API 共享
- **示例**: 测试 REST API、管理接口文档

### 5. shadcn/ui MCP Server
**服务**: `mcp7_*` 系列
- **作用**: shadcn/ui 组件库管理
- **使用场景**:
  - 添加 UI 组件到项目
  - 查看组件示例
  - UI 组件配置
- **示例**: 添加按钮组件、查看卡片示例

## 使用指南

### 何时使用技能
1. **项目构建问题** → 使用 `qt-build-and-fix`
2. **架构设计** → 使用 `qt-architecture`
3. **布局问题** → 使用 `qt-layouts`
4. **调试问题** → 使用 `qt-debugging`
5. **QML 开发** → 使用 `qt-qml`
6. **UI 设计** → 使用 `frontend-design`
7. **代码审查** → 使用 `code-reviewer`
8. **文档更新** → 使用 `documentation-update`

### 何时使用 MCP 服务
1. **需要外部信息** → 使用 Tavily 搜索
2. **需要库文档** → 使用 Context7
3. **需要 Web 测试** → 使用 Playwright
4. **需要 API 测试** → 使用 Postman API
5. **需要 UI 组件** → 使用 shadcn/ui

### 最佳实践
1. **优先使用项目特定技能**: Qt 相关问题优先使用 qt-* 系列技能
2. **合理使用 MCP 服务**: 需要外部资源时才使用 MCP
3. **技能组合使用**: 复杂问题可以组合多个技能
4. **及时更新文档**: 完成功能后使用 `documentation-update` 更新文档

### 注意事项
- 技能选择应基于具体问题类型
- MCP 服务使用需要网络连接
- 某些技能可能有特定前置条件
- 使用技能前应明确问题范围


```
用户: 完成了新功能开发
解决: 使用 documentation-update 技能更新相关文档
```
