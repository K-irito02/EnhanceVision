# 构建系统

## 构建基线

| 配置项 | 值 |
|--------|-----|
| 构建系统 | CMake 3.20+ |
| 生成器 | Visual Studio 17 2022 (x64) |
| C++ 标准 | C++20 |

## 构建预设

| 预设 | 用途 | 输出目录 |
|------|------|----------|
| `windows-msvc-2022-debug` | 开发调试 | `build/msvc2022/Debug` |
| `windows-msvc-2022-release` | 性能测试 | `build/msvc2022/Release` |

## 构建原则

1. **优先增量构建**：避免无故清理 `build/`
2. **变更 CMake 后重新配置**
3. **构建失败先修复根因**：不绕过警告/错误

## 部署原则

1. **发布前必须完成运行验证**
2. **Qt 运行时依赖通过 `windeployqt` 部署**
3. **打包步骤可复现**
4. **安装器改动必须验证完成页启动链路**：至少覆盖首启语言、拖拽导入、默认路径写入与设置页展示一致性
5. **涉及 Windows 提权启动时必须检查交互回归**：特别是 Explorer 到应用的拖拽、剪贴板和文件选择行为
6. **安装器升级改动必须验证首启数据一致性**：至少覆盖会话内容、缓存统计、窗口状态与真实生效数据目录一致
7. **发布产物必须生成并保留校验信息**：安装包和便携包都应输出 SHA256 校验文件，避免分发损坏包

## 日志要求

1. **默认运行日志只保留问题导向信息**：常规 `qInfo/qDebug` 不应留在高频热路径
2. **正常关闭链路不得使用 `WARN` 噪声日志**：仅真实异常、恢复失败或数据风险使用告警级别
3. **排查历史日志时区分“旧残留”与“当前问题”**：不要因为过期测试日志或历史异常退出记录误判当前状态
