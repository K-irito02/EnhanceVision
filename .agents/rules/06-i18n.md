# 国际化

## 语言范围

| 语言 | 代码 | 文件 |
|------|------|------|
| 默认 | `zh_CN` | `app_zh_CN.ts` |
| 支持 | `en_US` | `app_en_US.ts` |

## 文本规则

| 位置 | 方法 |
|------|------|
| QML | `qsTr()` |
| C++ | `tr()` |

1. **禁止硬编码用户可见文本**
2. **动态文本使用 `%N` 占位**：`qsTr("处理 %1 个文件").arg(count)`

## 更新流程

```powershell
# 1. 更新翻译源文件
cmake --build . --target EnhanceVision_lupdate

# 2. 编辑翻译（Qt Linguist）
linguist resources/i18n/app_zh_CN.ts

# 3. 生成二进制文件
cmake --build . --target EnhanceVision_lrelease
```

## UI 适配约束

1. **按钮最小宽度 80px**：容纳中英文本
2. **文本容器高度自适应**：禁止固定高度
3. **长文本设置 `elide`**：`elide: Text.ElideRight`
4. **功能完成后切换英文验证布局**

## Qt 基础翻译加载策略

1. `app_*.qm` 使用资源路径 `:/i18n/`
2. Qt 内置翻译优先从 `applicationDirPath()/translations` 加载
3. 若未命中，再回退 `QLibraryInfo::TranslationsPath`
4. 允许候选名回退：`qtbase_xx_YY` -> `qt_xx_YY` -> `qtbase_xx` -> `qt_xx`
5. 英文（`en*`）可不加载 Qt 翻译，避免无效告警
6. 首次启动若 `settings.ini` 尚未写入语言，允许回退读取安装器语言，保证安装向导与首启界面语言一致

## 国际化日志约束

1. 翻译切换流程避免输出冗余成功日志
2. 仅在真正失败且影响非英文语言时输出 `WARN`
3. 修复后必须检查 `logs/runtime_output.log` 中是否仍有持续性翻译告警
4. 涉及安装器语言或首启语言逻辑时，必须同时验证中文安装和英文安装两条路径
